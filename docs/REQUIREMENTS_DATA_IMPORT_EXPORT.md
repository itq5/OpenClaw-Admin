## 1. 功能需求描述

### 1.1 功能概述

数据导入导出功能为用户提供完整的数据备份和恢复能力，支持以下核心场景：

1. **数据导出**: 将系统内的所有数据（用户、任务、场景、Cron 任务等）导出为标准化格式（JSON + ZIP 压缩包）
2. **数据导入**: 从外部文件导入数据到系统，支持合并模式和替换模式
3. **历史记录**: 记录所有导入导出操作，支持历史追溯

### 1.2 功能范围

| 模块 | 导出支持 | 导入支持 | 说明 |
|-----|---------|---------|------|
| 用户数据 | ✅ | ✅ | 用户名、邮箱、角色、权限等 |
| 任务数据 | ✅ | ✅ | 任务标题、描述、状态、时间等 |
| 场景数据 | ✅ | ✅ | 场景名称、配置、关联任务等 |
| Cron 任务 | ✅ | ✅ | 任务定义、调度表达式、执行历史等 |
| 角色权限 | ✅ | ✅ | 角色定义、权限配置等 |
| 系统设置 | ✅ | ✅ | 主题、语言、时区等配置 |

### 1.3 数据格式规范

**导出文件格式**:
- 主格式：ZIP 压缩包
- 内部结构:
  ```
  backup-20260411-153000.zip
  ├── manifest.json        # 导出元数据（时间、版本、包含表）
  ├── users.json           # 用户数据
  ├── tasks.json           # 任务数据
  ├── scenarios.json       # 场景数据
  ├── crons.json           # Cron 任务数据
  ├── roles.json           # 角色数据
  └── settings.json        # 系统设置
  ```

**导入文件格式**:
- 支持 ZIP 格式（标准导出格式）
- 支持单文件 JSON 格式（部分数据导入）

### 1.4 导入模式

| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| **MERGE** | 合并模式，保留现有数据，仅导入新数据或更新冲突数据 | 数据迁移、增量备份恢复 |
| **REPLACE** | 替换模式，清空现有数据，完全替换为导入数据 | 完整备份恢复、环境克隆 |

## 3. 技术需求

### 3.1 后端 API 设计

**导出 API**:
```
POST /api/export/full
  - 请求体：{ format: 'zip' | 'json', tables: string[] }
  - 响应：文件流（application/zip）
  
POST /api/export/specific
  - 请求体：{ tables: string[], format: 'zip' | 'json' }
  - 响应：文件流
```

**导入 API**:
```
POST /api/import
  - 请求体：multipart/form-data (file: 备份文件)
  - 参数：mode: 'MERGE' | 'REPLACE'
  - 响应：{ success: boolean, created: number, updated: number, skipped: number, errors: [] }

GET /api/import/history
  - 响应：{ total: number, items: [{ id, type, mode, status, createdAt, ... }] }
```

### 3.2 性能要求

| 指标 | 要求 |
|-----|------|
| 导出 10000 条记录 | < 30 秒 |
| 导入 1000 条记录 | < 10 秒 |
| 并发导入/导出 | 最多 3 个同时进行的任务 |
| 最大文件大小 | 100MB |

### 3.3 安全要求

- [ ] 导入导出操作需要管理员权限
- [ ] 文件上传类型校验（仅允许 ZIP/JSON）
- [ ] 文件大小限制（100MB）
- [ ] 导入前数据验证，失败时完整回滚
- [ ] 审计日志记录所有操作
- [ ] 敏感数据脱敏（如密码字段）

### 3.4 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 文件格式错误 | 返回明确错误信息，提示正确格式 |
| 数据验证失败 | 显示详细错误列表，不执行导入 |
| 磁盘空间不足 | 提前检查，返回友好错误 |
| 导入冲突 | 根据模式处理（MERGE 跳过/更新，REPLACE 覆盖） |
| 超时 | 异步处理，返回任务 ID 供查询进度 |
