## 详细审查发现

### 🔴 高风险 (High)

#### 1. 系统命令执行风险
**位置**: `server/index.js:628-642`  
**问题**: npm 更新功能使用 `execSync` 执行系统命令，虽然做了字符过滤但仍存在潜在风险
```javascript
const sanitizedSpec = packageSpec.replace(/[^a-zA-Z0-9@.-]/g, '')
const output = execSync(`npm install -g ${sanitizedSpec}`, { ... })
```
**影响**: 如果过滤逻辑被绕过，攻击者可执行任意系统命令  
**修复建议**:
- 使用 npm API 而非命令行执行
- 添加更严格的白名单验证
- 在沙箱环境中执行
- 记录完整的命令执行审计日志

**风险等级**: 🔴 高风险  
**修复优先级**: P0 - 立即修复

---

#### 2. PTY 终端会话无命令限制
**位置**: `server/index.js:1447`  
**问题**: PTY 终端会话允许执行任意 shell 命令，无命令白名单限制
```javascript
const ptyProcess = pty.spawn(shell, [], { ... })
```
**影响**:  authenticated 用户可执行任意系统命令  
**修复建议**:
- 实现命令白名单机制
- 限制可执行命令范围
- 添加命令执行审计和监控
- 考虑使用受限 shell

**风险等级**: 🔴 高风险  
**修复优先级**: P0 - 立即修复

---

### 🟠 中风险 (Medium)

#### 3. 默认管理员凭据
**位置**: `.env.example`  
**问题**: 示例配置中包含默认管理员凭据
```
AUTH_USERNAME=admin
AUTH_PASSWORD=admin
```
**影响**: 如果用户未修改默认凭据，系统易受攻击  
**修复建议**:
- 移除示例配置中的默认密码
- 首次启动时强制修改密码
- 使用强密码策略

**风险等级**: 🟠 中风险  
**修复优先级**: P1 - 短期修复

---

#### 4. 会话存储未持久化
**位置**: `server/index.js:196`, `server/auth.js`  
**问题**: 会话存储在内存 Map 中，服务器重启后所有会话失效
```javascript
const sessions = new Map()
```
**影响**: 
- 用户体验差（重启后需重新登录）
- 无法实现会话管理功能
- 分布式部署困难

**修复建议**:
- 将会话存储到数据库
- 实现会话持久化
- 添加会话过期清理任务

**风险等级**: 🟠 中风险  
**修复优先级**: P1 - 短期修复

---

#### 5. 速率限制基于内存
**位置**: `server/auth.js:10-22`  
**问题**: 速率限制和暴力破解防护基于内存存储
```javascript
const loginAttempts = new Map()
const apiRateLimits = new Map()
```
**影响**: 服务器重启后防护失效，分布式环境无效  
**修复建议**:
- 使用 Redis 等外部存储
- 实现分布式速率限制

**风险等级**: 🟠 中风险  
**修复优先级**: P2 - 中期优化

---

#### 6. CORS 配置允许无 Origin 请求
**位置**: `server/index.js:96-97`  
**问题**: 允许无 Origin 的请求通过
```javascript
if (!origin) return callback(null, true)
```
**影响**: 可能绕过 CORS 保护  
**修复建议**:
- 明确指定允许的源
- 对无 Origin 请求进行额外验证

**风险等级**: 🟠 中风险  
**修复优先级**: P1 - 短期修复

---

### 🟡 低风险 (Low)

#### 7. 缺少 HTTPS 强制
**位置**: `server/index.js`  
**问题**: 未强制使用 HTTPS，敏感信息可能明文传输  
**修复建议**:
- 生产环境强制 HTTPS
- 使用 HSTS 头

**风险等级**: 🟡 低风险  
**修复优先级**: P2 - 中期优化

---

#### 8. 错误信息可能泄露敏感数据
**位置**: 多处 API 端点  
**问题**: 部分错误响应包含详细技术信息  
**修复建议**:
- 生产环境隐藏详细错误
- 使用通用错误消息
- 记录详细错误到服务端日志

**风险等级**: 🟡 低风险  
**修复优先级**: P2 - 中期优化

---

#### 9. 依赖项版本未锁定
**位置**: `package.json`  
**问题**: 部分依赖未锁定具体版本  
**修复建议**:
- 使用 `package-lock.json`
- 定期扫描依赖漏洞

**风险等级**: 🟡 低风险  
**修复优先级**: P2 - 中期优化
